## 1. Access Control & Data Hiding

Access modifiers determine the visibility of classes, variables, and methods. The primary goal of access control is **Security and Data Hiding** â€” preventing outside classes from directly manipulating a class's internal state.

### The Problem with Direct Access

If `num` is `public`, anyone can change it directly (e.g., `obj.num = 10;`). This bypasses any internal logic or validation your class might need, which is a **bad convention**.

### The Solution: `private` & Getters/Setters

When you mark internal variables as `private`, they cannot be accessed outside that specific file.

- **Video Example (`ArrayList`):** If you look at the internal implementation of Java's `ArrayList`, variables like `DEFAULT_CAPACITY` are marked as `private`. You cannot directly access or change the default capacity of an `ArrayList` from your code.
- **Best Practice:** Make instance variables `private` and provide `public` getter and setter methods. This allows you to control how the data is modified and accessed.

### Key Takeaways for Interviews

- **Default (No Modifier):** Also known as "package-private." It is only accessible within the same package.
- **`protected` Exception:** A `protected` member can be accessed in a different package only by a subclass, and specifically through an instance of that subclass â€” not the parent class.
- **`public`:** Use sparingly for variables. Usually reserved for methods you want accessible from anywhere (like the `main` method).

## 2. Packages in Java

Packages are essentially **folders** used to organize related classes. There are two types:
- **User-Defined** â€” folders you create
- **In-Built** â€” provided by Java

### Important In-Built Packages

| Package | Description |
|---|---|
| `java.lang` | Contains essential language features (e.g., `String`, `System`, basic math). **Imported automatically** â€” no explicit import needed. |
| `java.io` | Contains classes for input/output operations (reading/writing files). |
| `java.util` | Contains utility classes, primarily the **Collections Framework** (`ArrayList`, `LinkedList`, etc.). |
| `java.applet` & `java.awt` | Used for GUI development â€” mostly outdated for modern applications. |
| `java.net` | Used for network operations. |

> **Crucial Note:** `java.lang` is imported automatically by default in every Java program. You don't need to explicitly write `import java.lang.*;`.

## 3. The `Object` Class

Every single class you create in Java **indirectly inherits** from the `Object` class. It is the topmost root class in the Java class hierarchy. Even if you don't write `extends Object`, the compiler does it for you internally.

Because of this, every object you instantiate inherits several useful methods from the `Object` class.

### Important Methods of the `Object` Class *(Interview Focus)*

#### 1. `hashCode()`
- Returns a random **integer representation** of the object.
- **Common Misconception:** `hashCode()` does **not** return the actual memory address. It is an integer generated by an algorithm (often used in HashMaps).

#### 2. `equals()` vs. `==`

| | Description |
|---|---|
| `==` (Comparator) | Checks if two reference variables point to the **exact same object** in memory. |
| `.equals()` (Method) | Checks the **actual content/value** of the objects. |

**Example:** If you create two different object instances with the same data (`num = 34`):
- `obj1 == obj2` â†’ `false` (different memory locations)
- You can **override** `.equals()` to return `true` by comparing their `num` values.

#### 3. `instanceof` Operator
- Checks if an object is an instance of a specific class or subclass.
- Example:
  ```java
  System.out.println(obj instanceof ObjectDemo); // true
  ```

#### 4. `getClass()`
- Returns **metadata** about the class at runtime (stored in heap memory).
- **Key Detail:** You **cannot** override `getClass()` because it is marked as `final` in the internal `Object` class implementation.
